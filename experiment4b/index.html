Experiment 4 - Infinite Eyeballs

Description

This project creates an infinite terrain overtaken by eyeballs that watch the user's movement. This is what nightmares are made of.

Technical

The program uses procedural generation algorithms combined with unique hash values created by the user's world key in order to generate different terrain patterns and eyeball placements. Using trigonometry functions such as atan2(), sin(), and cos() as well as other variables like mouseX and mouseY, I found the angle that will determine where the user's mouse will be. Then by using that angle I was able to move the pupil in respect to the user's mouse position. I also used the hash function provided by the instructor to determine where the pre-existing eyeballs will be placed.

Reflection

This experiment was very successful for me since I was able to implement eyeballs that follow the player's movement with their pupils. I did have difficulty trying to figure out how to get the angle so that the pupil is looking at the player's mouse. If I had more time, I would like to add more ambiance to the terrain and perhaps add a misty effect to make the terrain more spooky.

Artist Statement

Infinity: I used the infinite extent of my world to create a haunting world taken over by monstrous eyeballs.

Variation in Space: As the audience browses my world, they'll come across numerous eyeballs scattered around a purple and black terrain.

Variations in Key: Different worlds will generate eyeballs at different locations as well as change the pattern of the terrain.

Signs of Life: The user is allowed to place their own eyeball by clicking on tiles that is not taken by an eyeball already.
